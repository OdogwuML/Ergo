# Ergo — Property Management System

> *From the Greek "ἔργον" — work, task, deed.*
> Because every tenancy agreement and rent payment should be verifiably true.

Ergo is a modern, blockchain-backed property management platform built for the Nigerian market. It gives landlords complete oversight of their property portfolio and tenants a seamless way to pay and manage their tenancy — with every lease and payment anchored to a decentralized, tamper-proof ledger.

---

## ✨ Features

### For Landlords
- **Portfolio Dashboard** — Overview of all buildings, occupancy rates, and revenue at a glance
- **Building & Unit Management** — Add properties, create units, and track each one individually
- **Tenant Invite Flow** — Invite tenants by email or phone; auto-generates a lease agreement on invite
- **Payment Tracking** — Immutable, append-only rent payment records across all properties
- **Maintenance Management** — View and update tenant maintenance requests per unit
- **Document Vault** — All lease agreements stored on decentralized storage, verifiable anytime
- **Export** — Download payment records as PDF or Excel

### For Tenants
- **Tenant Dashboard** — Quick view of tenancy, rent due, and payment history
- **Rent Payment** — Pay rent securely via Interswitch (card, transfer, USSD) in ₦ Naira
- **Lease Access** — View and download auto-generated lease agreement at any time
- **Maintenance Requests** — Submit and track repair requests for their unit
- **Payment Receipts** — Every payment is chain-verified and downloadable

### Powered by Web3 (Invisible to Users)
- **Auto-Generated Lease Agreements** — PDF leases generated by the backend on tenant invite, instantly stored on **Supabase Storage** (no manual upload ever needed)
- **Managed Service Wallet** — All blockchain interactions handled server-side; neither landlords nor tenants need a crypto wallet

## 🚀 Getting Started

### Prerequisites
- Go 1.22+
- A [Supabase](https://supabase.com) project
- A [Paystack](https://paystack.com) account
- A [Resend](https://resend.com) account (email)
- A [Termii](https://termii.com) account (SMS)

### 1. Clone the Repository

```bash
git clone https://github.com/OdogwuML/ergo.git
cd ergo
```

### 2. Set Up Environment Variables

Create a `.env` file in the root directory:

```bash
# Supabase
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# Interswitch
INTERSWITCH_CLIENT_ID=
INTERSWITCH_SECRET_KEY=
INTERSWITCH_MAC_KEY=

# Resend (Email)
RESEND_API_KEY=re_...

# Termii (SMS)
TERMII_API_KEY=...

# App
APP_URL=http://localhost:8080
PORT=8080
```

### 3. Run the Backend

```bash
cd tools
go mod tidy
go run cmd/server/main.go
```

The server starts at `http://localhost:8080`.

### 4. Open the Frontend

The Go server serves the `web/` folder as static files. Just visit `http://localhost:8080` in your browser.

## 🔑 API Overview

All API routes are under `/api`. Authenticated routes require a `Bearer` token in the `Authorization` header.

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `POST` | `/api/auth/signup` | ❌ | Register a new user |
| `POST` | `/api/auth/login` | ❌ | Log in and receive JWT |
| `POST` | `/api/auth/accept-invite` | ❌ | Accept tenant invitation |
| `GET` | `/api/invitations/verify` | ❌ | Verify an invite token |
| `POST` | `/api/webhooks/interswitch` | ❌ | Interswitch payment webhook |
| `GET` | `/api/dashboard/landlord` | ✅ landlord | Landlord dashboard stats |
| `GET` | `/api/dashboard/tenant` | ✅ tenant | Tenant dashboard data |
| `GET/POST` | `/api/buildings` | ✅ landlord | List / create buildings |
| `GET/PUT` | `/api/buildings/:id` | ✅ landlord | Get / update building |
| `GET/POST` | `/api/buildings/:id/units` | ✅ landlord | List / create units |
| `POST` | `/api/invitations` | ✅ landlord | Send tenant invite |
| `GET` | `/api/invitations` | ✅ landlord | List pending invitations |
| `POST` | `/api/payments/initialize` | ✅ tenant | Initialize a rent payment |
| `GET` | `/api/payments` | ✅ | Payment history |
| `POST/GET` | `/api/maintenance` | ✅ | Create / list maintenance requests |
| `PUT` | `/api/maintenance/:id/status` | ✅ landlord | Update request status |
| `POST/GET` | `/api/documents` | ✅ | Upload / list documents |

## 🔐 Security

- All routes are protected with Supabase JWT verification
- Row-Level Security (RLS) enforced at the database level
- Tenants can only access data for their own unit
- Payment records are immutable (append-only by design)
- All blockchain keys are server-side only — users are never exposed to Web3
